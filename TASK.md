# TASK: Интеграция FlexLayout и развертывание IDE-интерфейса управления воркспейсом

## КОНТЕКСТ ЗАДАЧИ
Необходимо перенести и развернуть продвинутую систему управления окнами на базе библиотеки `flexlayout-react` в целевой проект `react/my-app`. Интерфейс должен полностью воспроизводить возможности промышленной IDE, включая управление табами, всплывающие окна (Popouts), кастомные панели границ (Borders) и гибридную логику открытия JSON-файлов, протестированную ранее.

---

## 1. ПОДРОБНОЕ ТЕХНИЧЕСКОЕ ЗАДАНИЕ (БЕЗ СОКРАЩЕНИЙ)

### Задача 1.1: Установка и базовая конфигурация FlexLayout
1. **Зависимости:** Установи библиотеку `flexlayout-react` (версии v0.9.x) в каталог `react/my-app`.
2. ** Стили:** Импортируй базовые стили FlexLayout в главный файл приложения (`index.css` или `App.css`), включая темную тему (`dark.css` или кастомную тему, соответствующую текущему UI).
3. **Инициализация модели:** Создай первоначальную разметку воркспейса (`Layout.fromJSON(initialConfig)`), которая по умолчанию содержит:
   * Главный центральный табсет с вкладкой "Холст графа" и постоянной системной вкладкой "Просмотр JSON".

### Задача 1.2: Реализация гибридного открытия вкладок (стиль VS Code)
1. **Постоянная вкладка ("Открыть"):** 
   * При обычном левом клике на JSON-файл в списке или на ноду графа, фокус переводится на персистентную вкладку `"Просмотр JSON"`. 
   * Внутри этой вкладки обновляется содержимое Monaco Editor кодом выбранного файла. Новые вкладки при этом не плодятся.
2. **Динамические вкладки ("Открыть в новой вкладке"):**
   * При клике на файл колесиком мыши (middle-click) или при нажатии на контекстную кнопку "Открыть в новой вкладке", во FlexLayout динамически генерируется новый таб через `layoutModel.addTabToActiveTabSet`.
   * Имя вкладки формируется динамически на основе свойства `smart_name` файла. Внутри вкладки рендерится изолированный экземпляр Monaco Editor для редактирования именно этого файла. Таб имеет кнопку закрытия (крестик).

### Задача 1.3: Расширенный оконный менеджмент (по демо FlexLayout v0.9)
Реализуй расширенные интерактивные фичи для работы с табсетами:
1. **Кнопка `+` (Add Tab):** В заголовках табсетов активируй кнопку `+`. При нажатии на неё в текущий активный табсет добавляется пустая вкладка с дефолтным JSON-шаблоном для создания нового промпта/компонента с нуля.
2. **Переименование по двойному клику:** Используя метод `onRenderTab`, перехватывай двойной клик по заголовку любого таба и временно заменяй текст на текстовый инпут для динамического переименования вкладки.
3. **Float / Popout окна:** Включи встроенную поддержку выноса вкладок во внешние окна браузера (`Actions.createPopout`). Пользователь должен иметь возможность «отлепить» вкладку с Monaco Editor и перетащить её на второй монитор с сохранением реактивного состояния.

### Задача 1.4: Панели Границ (Borders Config)
Сконфигурируй и отобрази системные панели границ FlexLayout (`flexlayout__border`):
1. **Нижний бар (`border_bottom`):**
   * Создай нижнюю панель. Размести в ней вкладку "Terminal / Логи парсинга" с иконкой терминала. Панель должна сворачиваться и разворачиваться по клику на кнопку в нижнем баре.
2. **Правый бар (`border_right`):**
   * Размести боковую панель управления глобальными атрибутами приложения. Интегрируй в неё следующие элементы управления:
     * **Чекбокс "Детализировать изолированный граф":** Глобальный UI-тумблер управления декомпозицией React Flow.
     * **Селектор размера шрифта:** Выпадающий список (`Size small`, `Size medium`, `Size large`) для изменения масштаба текста интерфейса и Monaco Editor.
     * **Селектор тем оформления:** Выпадающий список переключения визуальных тем (`Alpha Light`, `vs-dark`, `Rounded`).

## 2. ПОШАГОВЫЙ ПЛАН РЕАЛИЗАЦИИ ДЛЯ ИИ-АГЕНТА

* **Шаг 1: Среда и Пакеты.** Перейди в директорию `react/my-app`, установи `flexlayout-react`. Пропиши импорт стилей layout-движка в проект.
* **Шаг 2: Архитектура Стора (Zustand).** Расширь состояние приложения: добавь экшены для генерации табов (`addTab`), переименования, смены темы и размера шрифта. Свяжи модель FlexLayout со стейтом, чтобы изменения структуры окон кэшировались.
* **Шаг 3: Верстка Воркспейса.** Напиши/обнови компонент, рендерящий `<Layout model={model} factory={factory} />`. Внутри `factory` пропиши условия для рендеринга "Холста графа", "Терминала" и кастомных вкладок Monaco Editor.
* **Шаг 4: Интеграция Бордеров и Кастомного UI.** Добавь в конфигурацию JSON-модели описание `borders: [ { direction: "bottom", ... }, { direction: "right", ... } ]`. Наполни правый бар селекторами и чекбоксами, настроив их двустороннее связывание с Zustand-стором.
* **Шаг 5: Проверка Интерактивности.** Реализуй обработчики middle-click на элементах списков, протестируй открытие Monaco во временной и постоянных вкладках, проверь работу функции `Actions.createPopout()`.
