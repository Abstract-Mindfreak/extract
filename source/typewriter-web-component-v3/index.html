TypeWriter Web Component V3

A web component for character-by-character text animation with HTML support. Now featuring seek functionality for precise navigation through the animation timeline.

Core Features

  • HTML Support — Preserves tags including emphasis, links, code, and nested structures
  • Bidirectional Text — Supports LTR and RTL text direction via dir attribute
  • Dynamic Content — Update content with setText() method
  • Playback Control — Methods for start, pause, resume, complete, and reset
  • Progress Events — Dispatches progress events during animation
  • Accessibility Features — ARIA live regions and optional reduced motion support

Configuration Attributes

  • speed — Characters per second (default: 100)
  • min-duration — Minimum animation duration in ms (default: 50)
  • max-duration — Maximum animation duration in ms (default: 500)
  • autostart — Auto-start on load, set to "false" to disable (default: true)
  • dir — Text direction: "ltr" or "rtl" (default: ltr)
  • respect-motion-preference — Respect prefers-reduced-motion media query (default: false)
  • aria-label — ARIA label for the container region

JavaScript API

  • start() — Begin animation from start
  • pause() — Pause animation
  • resume() — Resume paused animation
  • complete() — Display all content instantly
  • reset() — Clear content and reset state
  • setText(html) — Replace content and reset
  • seek(position) — Seek to normalized position (0–1)
  • seekToPercent(percent) — Seek to percentage (0–100)
  • seekToChar(charIndex) — Seek to specific character
  • getProgress() — Get current progress state

Events

  • start — Dispatched when animation begins
  • pause — Dispatched when animation pauses
  • resume — Dispatched when animation resumes
  • complete — Dispatched when animation finishes
  • reset — Dispatched when component resets
  • progress — Dispatched every 2% with {current, total, percent, position}
  • seek — Dispatched when seeking with position details

Nested Lists

Parent tracking works across arbitrary nesting depth. Child lists animate in sequence as their parent item is reached.

  • Rendering
    • Light DOM — inherits page styles
    • No Shadow DOM — no style isolation
  • Animation model
    • Character-by-character via setTimeout
    • Node map rebuilt deterministically on each seek
    • Generation counter (_gen) prevents stale callbacks
  • Events
    • Progress — 2% threshold delta
    • Seek — position + canResume flag

Code Block Support

Whitespace inside <pre> is normalized like all other text nodes. Useful for inline API examples.

const el = document.querySelector("type-writer");

el.addEventListener("progress", (ev) => {
  console.log(ev.detail.percent.toFixed(1) + "%");
});

el.seek(0.5);
el.resume();

Definition Lists

Arbitrary block elements animate correctly — the component does not assume list type, only walks child nodes recursively.

position
Normalized float from 0 to 1 representing current progress through the full character count.
current
Number of characters rendered so far in the current animation pass.
total
Total character count for the current content, computed once during flatten and cached.
percent
Convenience alias for position * 100, always consistent with the position value.

Inline Nesting

Deeply nested inline elements — bold italic, linked bold, bold italic code — all animate without breaking the parent chain. Each character is appended to its correct immediate parent regardless of nesting depth.

Technical Implementation

Built using Custom Elements API with Light DOM for CSS inheritance. The seek functionality uses DOM state reconstruction with node mapping to maintain parent-child relationships. This ensures seeking works seamlessly with pause/resume operations without breaking the animation flow or skipping content. Whitespace normalization collapses consecutive spaces and supports all modern browsers with Custom Elements.
