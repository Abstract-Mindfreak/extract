# MMSS React - Руководство по работе приложения от начала до конца

## Цель документа

Данный документ демонстрирует полный рабочий процесс приложения от запуска до получения готового JSON для отправки в musicflow.app. Оценивается удобство UI, функциональность, количество шагов и наглядность каждого режима.

**Исходные условия:**
- Prompt Library уже содержит блоки
- Сессии импортированы с 4 аккаунтов
- Session blocks присутствуют в системе

---

## Обзор приложения

Приложение MMSS React состоит из 5 основных вкладок:

1. **Performance** - Компактный перформанс режим для live-выступлений
2. **Advanced** - Расширенные контролы для детальной настройки
3. **Prompt Library** - Библиотека промптов и сборка JSON для musicflow.app
4. **ASE Console** - Консоль ASE с 10 режимами управления
5. **Archives** - Архивы сессий и треков

**Основной путь к готовому JSON:** ASE Console → Prompt Library → JsonSequenceBuilder → Экспорт

## Вкладка 1: Performance

### Назначение
Компактный режим для live-перформанса с минимальным набором контролов.

### UI Элементы

#### CompactTransportBar
- **Play/Pause** - Запуск/остановка воспроизведения аудио
- **BPM** - Настройка темпа (влияет на скорость всех процессов)
- **Volume** - Громкость мастера
- **Scene Select** - Выбор сцены (BASELINE, AURORA_GATE, GLASS_ORBIT, NOISE_TIDE)

#### StageCanvas
- Визуализация аудио в реальном времени
- Отображение осциллографа
- Поддержка drag-and-drop изображений для анализа

#### Pattern Strip (MiniMatrixStrip)
- Компактное отображение матрицы паттернов
- Кнопка "Open Advanced" для перехода в Advanced режим

#### Orbit Pad
- 4 пресета: Calm, Intense, Chaos, GlassOrbit
- Клик на пресет применяет настройки
- Long-press сохраняет текущее состояние в слот

#### Quick Status
- Отображение последнего чекпоинта
- Последний intent
- Скорость орбиты
- Контраст изображения

### Шаги для работы в Performance

1. **Запуск аудио**
   - Клик "Play" (1 клик)
   - Аудио движок инициализируется автоматически

2. **Настройка темпа**
   - Клик на BPM поле (1 клик)
   - Ввод значения (клики зависят от значения)

3. **Выбор сцены**
   - Клик на dropdown сцены (1 клик)
   - Выбор сцены (1 клик)

4. **Применение орбитального пресета**
   - Клик на пресет (1 клик)

**Итого: 4-6 кликов для базовой настройки**

- **Удобство:** ⭐⭐⭐⭐⭐ (5/5) - максимально компактно

## Вкладка 3: Prompt Library ⭐

### Назначение
**ОСНОВНОЙ РЕЖИМ ДЛЯ ГЕНЕРАЦИИ JSON** - библиотека промптов, визуальный редактор логики, сборка последовательностей и экспорт готового JSON для musicflow.app.

### UI Элементы

#### Top Bar
- **Load Library** - Загрузка библиотеки промптов (disabled если уже загружена)
- **Import Session Blocks** - Импорт блоков из сессий архивов
- **Reset Panel Layout** - Сброс расположения панелей к дефолтному
- **Clear Active Composition** - Очистка текущей активной композиции

#### Sidebar (Sections)
- **Block Library** (JsonBlockList) - Каталог блоков с поиском и фильтрами
- **Blockly** (PromptLogicBlocklyPanel) - Визуальный DSL для логики
- **Block Editor** (JsonBlockEditor) - Редактор метаданных и JSON блока
- **Sequences** (JsonSequenceBuilder) - Сборка последовательностей и экспорт
- **Bindings** (JsonBindingsPanel) - Матрица биндингов 4x4

#### Panel Move Actions
- Стрелки ←→↑↓ для перемещения панелей в сетке

---

### Панель 1: Block Library (JsonBlockList)

#### UI Элементы
- **Search Input** - Поиск по названию, описанию, тегам
- **Category Filter** - Фильтр по категории (all, ase_mode, manual, и т.д.)
- **Tag Filter** - Фильтр по тегам
- **Source Filter** - Фильтр по источнику (all, flowmusic.app, manual)
- **Block Cards** - Карточки блоков с:
  - Цветовая точка (ui.color)
  - Название блока
  - Категория
  - Описание
  - Source pill (источник: flowmusic.app/manual)
  - Conversation ID pill (если есть)
  - Track ID pill (если есть)
  - Tool Name pill (если есть)
  - Теги
  - Кнопки: Duplicate, Delete, Bind

#### Кнопки на карточке блока
- **Duplicate** - Создать копию блока
- **Delete** - Удалить блок
- **Bind** - Подготовить биндинг к кнопке (переключает в Bindings панель)

### Шаги для работы с Block Library

1. **Поиск блока**
   - Ввести текст в поиск (клики зависят от текста)
   - Или выбрать фильтр категории (1 клик)
   - Или выбрать фильтр тега (1 клик)

2. **Выбор блока**
   - Клик на карточку блока (1 клик)
   - Блок открывается в Block Editor

3. **Дублирование блока**
   - Клик "Duplicate" на карточке (1 клик)

4. **Удаление блока**
   - Клик "Delete" на карточке (1 клик)

**Итого: 1-3 клика для базовых операций**

### Оценка удобства
- **Удобство:** ⭐⭐⭐⭐⭐ (5/5) - отличный поиск и фильтры
- **Наглядность:** ⭐⭐⭐⭐⭐ (5/5) - цветовая кодировка, понятные метки
- **Интуитивность:** ⭐⭐⭐⭐⭐ (5/5) - drag-and-drop, понятные кнопки
- **Функциональность для JSON:** ✅ Критически важен - источник блоков для композиции

### Панель 2: Blockly (PromptLogicBlocklyPanel)

#### UI Элементы
- **Workspace** - Визуальная область для сборки блоков логики
- **Toolbox** - Панель с блоками:
  - **MMSS Logic** - mmss_prompt_condition, mmss_add_block, mmss_use_sequence, mmss_set_merge_strategy
  - **Logic** - controls_if, logic_compare, logic_operation, logic_boolean
- **Context Preset** - Выбор пресета контекста (neuro_dark, industrial, и т.д.)
- **Source JSON** - Отображение исходного JSON
- **Run** - Запуск логики (генерация композиции)
- **Clear** - Очистка активной композиции
- **Expand/Collapse** - Развернуть/свернуть панель

#### Блоки MMSS Logic
- **mmss_prompt_condition** - Условие для применения логики
- **mmss_add_block** - Добавить блок в композицию
- **mmss_use_sequence** - Использовать сохраненную последовательность
- **mmss_set_merge_strategy** - Установить стратегию слияния (concat, merge_shallow, merge_deep)

### Шаги для работы с Blockly

1. **Выбор пресета контекста**
   - Клик на dropdown (1 клик)
   - Выбор пресета (1 клик)

2. **Сборка логики**
   - Перетащить блок из toolbox в workspace (1 drag)
   - Соединить блоки (клики зависят от сложности)
   - Настроить параметры блоков (клики зависят от количества параметров)

3. **Запуск логики**
   - Клик "Run" (1 клик)
   - Композиция генерируется и передается в JsonSequenceBuilder

**Итого: 3-10+ кликов в зависимости от сложности логики**

### Оценка удобства
- **Удобство:** ⭐⭐⭐ (3/5) - визуально хорошо, но требует понимания Blockly
- **Наглядность:** ⭐⭐⭐⭐ (4/5) - визуальное программирование понятно
- **Интуитивность:** ⭐⭐⭐ (3/5) - нужно изучить блоки
- **Функциональность для JSON:** ✅ Важен для автоматической сборки композиций

### Панель 3: Block Editor (JsonBlockEditor)

#### UI Элементы
- **Metadata Form** - Форма метаданных:
  - **Name** - Название блока
  - **Description** - Описание
  - **Category** - Категория
  - **Tags** - Теги (через запятую)
  - **Color** - Цвет (hex или выбор из палитры)
  - **Icon** - Иконка
- **JSON Editor** - Редактор JSON payload:
  - **Type** - Тип payload (flowmusic.app_prompt, producer.ai_prompt)
  - **Version** - Версия
  - **Data** - Данные в формате JSON
- **Кнопки:**
  - **Save** - Сохранить блок
  - **Format** - Форматировать JSON
  - **Export Payload** - Экспортировать только payload в буфер/файл
  - **Export Block** - Экспортировать весь блок

### Шаги для работы с Block Editor

1. **Редактирование метаданных**
   - Изменить название (клики зависят от текста)
   - Изменить описание (клики зависят от текста)
   - Выбрать категорию (1 клик)
   - Добавить теги (клики зависят от количества тегов)

2. **Редактирование JSON**
   - Изменить type (1 клик)
   - Изменить version (клики зависят от текста)
   - Редактировать data (клики зависят от изменений)

3. **Сохранение**
   - Клик "Save" (1 клик)

4. **Экспорт**
   - Клик "Export Payload" или "Export Block" (1 клик)

**Итого: 3-10+ кликов**

### Оценка удобства
- **Удобство:** ⭐⭐⭐⭐ (4/5) - понятная форма, валидация JSON
- **Наглядность:** ⭐⭐⭐⭐ (4/5) - цветовая подсветка JSON
- **Интуитивность:** ⭐⭐⭐⭐ (4/5) - стандартные поля формы
- **Функциональность для JSON:** ✅ Важен для редактирования блоков

### Панель 4: Sequences (JsonSequenceBuilder) ⭐⭐⭐

#### UI Элементы

##### Saved Sequences (левая колонка)
- **Source Filter** - Фильтр по источнику (all, flowmusic.app, manual)
- **Sequence Cards** - Карточки сохраненных последовательностей:
  - Название
  - Source pill (источник)
  - Conversation ID pill
  - Tracks count pill
  - Merge Strategy
  - Клик загружает последовательность в активную композицию

##### Active Composition (центр)
- **Composition Blocks** - Список блоков в текущей композиции:
  - Порядок (order)
  - Название блока
  - Кнопки: Remove, Reorder (drag)
- **Add Block** - Dropdown для добавления блока из библиотеки
- **Merge Strategy** - Выбор стратегии слияния:
  - concat - конкатенация
  - merge_shallow - поверхностное слияние
  - merge_deep - глубокое слияние
- **External Pipeline Banner** - Отображается если композиция загружена из ASE Console

##### Actions (правая колонка)
- **Generate Preset** - Генерация пресет композиции:
  - Preset Count - количество блоков
  - Mode - режим (random, ordered, tag_weighted)
  - Кнопка "Generate"
- **Save as Sequence** - Сохранение текущей композиции:
  - Name - название
  - Description - описание
  - Кнопка "Save"
- **Preview JSON** - Предпросмотр объединенного JSON:
  - Кнопка "Show Preview"
  - Отображение результата слияния
  - Кнопка "Copy Preview" - копировать в буфер
  - Кнопка "Save Preview" - сохранить в файл
- **Batch Export** - Пакетный экспорт:
  - Formula - формула (например, "6x44 random")
  - Mode - режим
  - Tag Preset - пресет тегов
  - Кнопка "Export"

##### External Pipeline Banner (если загружено из ASE Console)
- Отображает источник композиции
- Количество modes
- Merge Strategy
- Порядок модулей

**Итого: 6 + (2 × N блоков) кликов**

3. **Предпросмотр JSON**

4. **Копирование JSON**

**Итого: 3 клика в ASE Console + 2 клика в Prompt Library = 5 кликов**

#### Tab: Control
Отображает выбранный режим (variation)

#### Tab: Formulas
Отображает справочник формул (FORMULAS)

#### Tab: Settings
Настройки консоли

### Режимы ASE Console

### Режим 1: ASE Unified Console ⭐⭐⭐⭐⭐

#### Назначение
Комбинированная консоль со всеми режимами в виджет-системе. Позволяет выбрать несколько модулей, настроить их параметры и отправить в JsonSequenceBuilder.

#### UI Элементы (Виджет-система)

**Виджет-система** - 11 перемещаемых и сворачиваемых виджетов:

1. **STATS_WIDGET** - Метрики системы
   - Модулей в rack
   - Velocity
   - Negentropy
   - Phi Sync

2. **MODULE_LIBRARY** - Библиотека модулей
   - Search Input - Поиск по названию, тегу, описанию
   - Checkbox "Только выбранные" - Фильтр
   - SELECT ALL - Выбрать все
   - CLEAR - Очистить выбор
   - Module Cards - Карточки с названием, категорией, описанием, тегами
   - Кнопка SELECT/REMOVE - Добавить/удалить из rack

3. **PIPELINE_RACK** - Rack выбранных модулей
   - Drag-and-drop для изменения порядка
   - Порядок влияет на merge pipeline
   - COPY JSON - Копировать в буфер
   - SEND TO BUILDER ⭐ - Отправить в JsonSequenceBuilder

4. **ENTROPY_PAD** - Визуальный pad для настройки Entropy P/C
   - 2D pad с курсором
   - Metric Chips для значений

5. **QUICK_CONTROLS** - Быстрые контролы
   - Q Gravity - слайдер
   - Phase - dropdown
   - LFE Mode - dropdown

6. **LOGIC_AND_META** - Логика и мета-параметры
   - Meta key - ввод текста
   - Op Mode кнопки

7. **PROMPT_LIBRARY** - Блоки из Prompt Library
   - Metric Chip "Загружено"
   - Search Input
   - Block Cards с кнопкой "+" для выбора

8. **ARCHIVE_TRACKS** - Треки из архивов
   - Metric Chip "Всего"
   - Search Input
   - Track Cards с кнопкой "+" для выбора

9. **CONFIGURATION_DB** - Сохранение конфигураций (свернут по умолчанию)
   - Config Name
   - SAVE кнопка
   - Saved Configs список

10. **JSON_PREVIEW** - Предпросмотр JSON
    - SHOW/HIDE кнопка
    - Отображение JSON

11. **SYSTEM_EVENT_FEED** - Лог событий (свернут по умолчанию)
    - Список событий
    - Preview stream

**Управление виджетами:**
- Кнопка ↑↓ - перемещение виджета вверх/вниз
- Кнопка chevron - сворачивание/разворачивание виджета
- Hover эффекты для наглядности

#### Шаги для работы в ASE Unified Console

1. **Выбор модулей**
   - Поиск модуля (клики зависят от текста)
   - Клик checkbox на карточке (1 клик × N модулей)
   - Или клик "SELECT ALL" (1 клик)

2. **Настройка порядка модулей**
   - Drag-and-drop в PIPELINE_RACK (1 drag × N перестановок)
   - Порядок критически важен для merge pipeline

3. **Настройка параметров**
   - Использовать QUICK_CONTROLS (1-3 клика)
   - Использовать ENTROPY_PAD (drag)
   - Использовать LOGIC_AND_META (клики зависят от ввода)

4. **Отправка в JsonSequenceBuilder**
   - Клик "SEND TO BUILDER" в PIPELINE_RACK (1 клик)
   - Автоматическое переключение на Prompt Library → JsonSequenceBuilder

5. **В JsonSequenceBuilder**
   - Композиция уже загружена
   - Клик "Show Preview" (1 клик)
   - Клик "Copy Preview" (1 клик)

**Итого: 3 + (1 × N модулей) + (1 × N перестановок) + 2 = 5+ кликов**

### Оценка удобства ASE Unified Console
- **Удобство:** ⭐⭐⭐⭐⭐ (5/5) - виджет-система с перемещением и сворачиванием
- **Наглядность:** ⭐⭐⭐⭐⭐ (5/5) - отличная визуализация, понятные иконки
- **Интуитивность:** ⭐⭐⭐⭐⭐ (5/5) - понятные кнопки ↑↓, hover эффекты
- **Функциональность для JSON:** ✅✅✅ КРИТИЧЕСКИ ВАЖЕН - основной путь для сложных композиций
